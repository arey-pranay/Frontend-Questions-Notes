# 📄 `call`, `apply`, `bind`, `this` — Complete Frontend Interview Notes

---

# 🧠 First Understand `this`

`this` is determined by **how a function is called**, not where it is defined (except arrow functions).

---

## Global Invocation

```js
function hello() {
  console.log(this);
}

hello();
```

Browser:

```js
window
```

Strict mode:

```js
undefined
```

---

## Method Invocation

```js
const user = {
  name: "Pranay",

  greet() {
    console.log(this.name);
  }
};

user.greet();
```

Output:

```js
Pranay
```

Because:

```text
this = object before the dot
```

---

# 🚨 Common Interview Trap

```js
const user = {
  name: "Pranay",

  greet() {
    console.log(this.name);
  }
};

const fn = user.greet;

fn();
```

Output:

```js
undefined
```

because:

```text
method detached from object
```

Now it's just a normal function call.

---

# `call()`

Invokes immediately.

Syntax:

```js
fn.call(thisArg, arg1, arg2, arg3);
```

---

Example:

```js
function greet(city) {
  console.log(this.name, city);
}

greet.call(
  { name: "Pranay" },
  "Indore"
);
```

Output:

```js
Pranay Indore
```

---

# `apply()`

Same as call.

Only difference:

Arguments are passed as array.

---

```js
fn.apply(thisArg, [arg1, arg2]);
```

---

Example

```js
greet.apply(
  { name: "Pranay" },
  ["Indore"]
);
```

---

# Difference

---

### call

```js
fn.call(obj, 1, 2, 3);
```

---

### apply

```js
fn.apply(obj, [1, 2, 3]);
```

---

# `bind()`

Most important.

---

Unlike call/apply:

```text
Does NOT execute
```

Returns a NEW function.

---

Example

```js
const boundFn =
  greet.bind(
    { name: "Pranay" },
    "Indore"
  );

boundFn();
```

Output:

```js
Pranay Indore
```

---

# Mental Model

```js
bind()
```

=

```text
Create new function
with permanently fixed this
```

---

# Comparison Table

| Method | Executes Immediately | Returns Function |
| ------ | -------------------- | ---------------- |
| call   | ✅                    | ❌                |
| apply  | ✅                    | ❌                |
| bind   | ❌                    | ✅                |

---

# Visual Memory Trick

```text
call  → Call now
apply → Apply now
bind  → Bind for later
```

---

# Partial Application

`bind` can pre-fill arguments.

---

Example

```js
function add(a, b) {
  return a + b;
}

const add5 =
  add.bind(null, 5);

add5(10);
```

Output:

```js
15
```

---

Internally:

```js
add(5, 10)
```

---

# Event Handler Problem

Very common.

---

```js
class User {

  constructor() {
    this.name = "Pranay";
  }

  click() {
    console.log(this.name);
  }
}
```

---

```js
button.onclick = user.click;
```

May lose correct `this`.

---

Fix:

```js
button.onclick =
  user.click.bind(user);
```

---

# React Class Components

Old React interviews love this.

---

```js
class App extends React.Component {

  constructor() {
    super();

    this.handleClick =
      this.handleClick.bind(this);
  }
}
```

---

Reason:

```text
event system loses this
```

# Arrow Functions Change Everything

---

Arrow functions DO NOT create their own `this`.

---

```js
const obj = {
  name: "Pranay",

  fn: () => {
    console.log(this.name);
  }
};
```

Output:

```js
undefined
```

---

Because arrow function uses:

```text
lexical this
```

from surrounding scope.

---

# Normal Function

```js
const obj = {
  name: "Pranay",

  fn() {
    console.log(this.name);
  }
};
```

Output:

```js
Pranay
```

---

# 🚨 Huge Interview Question

Can call/apply/bind change arrow function this?

---

NO.

---

Example

```js
const fn = () => {
  console.log(this);
};

fn.call({ name: "Pranay" });
```

Still ignores provided object.

---

Because:

```text
arrow this is permanently lexical
```

---

# Constructor + bind

Interesting behavior.

---

```js
function Person(name) {
  this.name = name;
}

const BoundPerson =
  Person.bind(null);

new BoundPerson("Pranay");
```

Works.

---

Because:

```text
new overrides bind's this
```

---

# `this` Priority Rules

Most interviewers ask this.

---

Priority order:

```text
new
↓
call/apply/bind
↓
object.method()
↓
default/global
```

---

Example

```js
new Fn()
```

beats:

```js
Fn.call(...)
```

---

# `arguments` vs Rest Parameters

Old JS:

```js
function x() {
  console.log(arguments);
}
```

---

Modern:

```js
function x(...args) {
  console.log(args);
}
```

---

Difference:

| arguments        | rest              |
| ---------------- | ----------------- |
| array-like       | real array        |
| old              | modern            |
| no array methods | has array methods |

---

# Why `apply()` Is Less Common Today

Before spread:

```js
Math.max.apply(
  null,
  arr
);
```

---

Modern JS:

```js
Math.max(...arr);
```

---

Spread replaced many apply usages.

# Polyfill Interview Questions

Very common:

Implement:

```js
Function.prototype.myCall
Function.prototype.myApply
Function.prototype.myBind
```

---

Core idea:

Temporarily attach function to object.

---

Example

```js
obj.temp = fn;
obj.temp();
delete obj.temp;
```

This is how `call()` can be polyfilled.

```js
function outer() {
  let count = 0;

  return function() {
    count++;
  };
}
```

Inner function remembers outer variables.

---

## Prototype Methods

```js

```

All functions inherit these.

Because:

```text
functions are objects
```
