# 📄 JavaScript Spread, Rest & Related Syntax — Frontend Interview Revision

These are among the most commonly used modern JavaScript features in:

* React
* Redux
* Next.js
* APIs
* state updates
* function arguments
* immutability patterns

Interviewers expect deep understanding here.

---

# 1️⃣ Spread Operator `...`

## ✅ Purpose

Spread EXPANDS values.

---

# Arrays

```js id="l11"
const arr = [1, 2, 3];

console.log(...arr);
```

Output:

```text id="l12"
1 2 3
```

---

# Objects

```js id="l13"
const obj = {
  name: "Pranay"
};

const clone = {
  ...obj
};
```

---

# Function Arguments

```js id="l14"
Math.max(...arr);
```

---

# 🚨 Mental Model

```text id="l15"
Spread = unpack/open values
```

---

# 2️⃣ Rest Operator `...`

## ✅ Purpose

Rest COLLECTS values.

---

# Function Parameters

```js id="l16"
function sum(...nums) {
  return nums.reduce((a, b) => a + b);
}
```

`nums` becomes array.

---

# Object Destructuring

```js id="l17"
const obj = {
  a: 1,
  b: 2,
  c: 3
};

const { a, ...rest } = obj;
```

Output:

```js id="l18"
rest = {
  b: 2,
  c: 3
}
```

---

# 🚨 Mental Model

```text id="l19"
Rest = gather/collect remaining values
```

---

# 🔥 EASIEST MEMORY TRICK

```text id="l20"
Spread expands
Rest collects
```

Same syntax:

* different meaning based on context

---

# 3️⃣ Array Spread

---

# Cloning Arrays

```js id="l21"
const copy = [...arr];
```

---

# Merging Arrays

```js id="l22"
const merged = [...a, ...b];
```

---

# Adding Elements

```js id="l23"
const updated = [...arr, 4];
```

---

# 🚨 Important

Spread creates:

* shallow copy

NOT deep copy.

---

# Example

```js id="l24"
const arr = [{ x: 1 }];

const copy = [...arr];

copy[0].x = 99;
```

Original also changes.

# 🚨 Property Override Order

Later properties win.

```js id="l27"

}
```

Result:

```js id="l28"

```

# 7️⃣ Shallow vs Deep Copy

VERY IMPORTANT.

---

# Spread = shallow copy

Nested objects still shared.

---

# Example

```js id="l39"
const obj = {
  nested: {
    x: 1
  }
};

const copy = { ...obj };

copy.nested.x = 99;
```

Original changes too.

---

# 🚨 Why?

Only top-level copied.

Nested references reused.

---

# Deep Copy Options

---

# `structuredClone`

Modern API.

```js id="l40"
structuredClone(obj)
```

---

# JSON Method

```js id="l41"
JSON.parse(JSON.stringify(obj))
```

Limitations:

* loses functions
* loses Dates
* loses undefined
