* Deep clone

* Polyfills
* React state updates

* Tree problems

# Part 1 — Traversal Decision Tree

Whenever you see:

```js
unknown input
```

your mental flow should be:

```text
null ?
↓
Array ?
↓
Object ?
↓
Primitive ?
```

Typical pattern:

```js
if (value === null) {
}

else if (Array.isArray(value)) {
}

else if (typeof value === "object") {
}

else {
}
```

---

# Why This Order?

Because:

```js
typeof []
// "object"
```

and

```js
typeof null
// "object"
```

---

# Wrong

```js
if (typeof value === "object") {
}
else if (Array.isArray(value)) {
}
```

Arrays never reach second condition.

# Part 2 — Every JS Type Check

---

## typeof

Used for primitives.

```js
typeof 5
// "number"

typeof "abc"
// "string"

typeof true
// "boolean"

typeof undefined
// "undefined"

typeof Symbol()
// "symbol"

typeof 1n
// "bigint"
```

---

## Functions

```js
typeof (()=>{})
// "function"
```

Special case.

---

## Objects

```js
typeof {}
// "object"
```

---

## Arrays

```js
typeof []
// "object"
```

Not useful.

Use:

```js
Array.isArray(arr)
```

---

## Null

Famous bug.

```js
typeof null
// "object"
```

Always remember.

# Part 3 — Array Traversals

---

## for...of

Most common.

```js
for (const item of arr) {
}
```

Gives:

```js
value
```

---

Example

```js
["a","b","c"]
```

↓

```js
a
b
c
```

---

## Need Index?

```js
for (const [i, value] of arr.entries()) {
}
```

---

Example

```js
0 a
1 b
2 c
```

---

# Classic For Loop

```js
for(let i=0;i<arr.length;i++)
```

Use when:

* index needed
* mutation needed
* performance critical

---

# forEach

```js
arr.forEach((value,index)=>{
})
```

Cannot:

```js
break
continue
return outer function
```

---

# map

Transforms.

```js
const doubled =
arr.map(x=>x*2);
```

---

# filter

Keeps elements.

```js
arr.filter(x=>x>5);
```

---

# reduce

Accumulates.

```js
arr.reduce(...)
```

---

# some

Any match?

```js
arr.some(...)
```

---

# every

All match?

```js
arr.every(...)
```

---

# find

Returns first matching element.

```js
arr.find(...)
```

---

# findIndex

Returns index.

```js
arr.findIndex(...)
```

# Part 4 — Object Traversals

---

# Object.keys()

Most common.

```js
Object.keys(obj)
```

Returns:

```js
["name","age"]
```

---

Traverse:

```js
for(const key of Object.keys(obj))
```

---

# Object.values()

Returns:

```js
["Pranay",23]
```

---

# Object.entries()

Returns:

```js
[
 ["name","Pranay"],
 ["age",23]
]
```

---

Best pattern:

```js
for(const [key,value]
    of Object.entries(obj))
{
}
```

---

# for...in

Dangerous.

```js
for(const key in obj)
```

Traverses:

```text
Own Properties
+
Inherited Properties
```

---

Example

```js
const parent = {
  x:1
};

const child =
Object.create(parent);

child.y = 2;
```

---

Output

```js
for(const k in child)
```

↓

```text
y
x
```

---

Safer:

```js
if(Object.hasOwn(obj,key))
```

---

# Part 5 — Object.hasOwn

Checks:

```text
Direct property?
```

---

Example

```js
Object.hasOwn(
  obj,
  "name"
)
```

---

Equivalent old style:

```js
obj.hasOwnProperty("name")
```

but safer.

---

# Why Safer?

Because object may override:

```js
{
  hasOwnProperty: null
}
```

---

# Part 6 — Recursive Traversal Pattern

Most interview problems:

```text
Flatten
Deep Clone
Camel Case
Resolve Functions
```

all use:

```js
function traverse(value){

  if(base_case){
    return value;
  }

  if(Array.isArray(value)){
    ...
  }

  if(typeof value==="object"){
    ...
  }
}
```

---

# Recursive Array Traversal

```js
for(const item of arr){
  traverse(item);
}
```

---

# Recursive Object Traversal

```js
for(const [k,v]
    of Object.entries(obj))
{
  traverse(v);
}
```

---

# Part 7 — Property Enumerability

Objects have:

```text
Enumerable
Non-enumerable
```

properties.

---

Visible:

```js
Object.keys()
```

only returns enumerable keys.

---

Example

```js
Object.defineProperty(
  obj,
  "hidden",
  {
    enumerable:false
  }
);
```

Won't appear.

---

# Part 8 — Reflect.ownKeys()

Most complete traversal.

Returns:

```text
String Keys
+
Symbol Keys
+
Non-enumerable Keys
```

---

Example

```js
Reflect.ownKeys(obj)
```

---

Advanced interview topic.

# Part 9 — Symbols

---

Normal:

```js
Object.keys(obj)
```

ignores:

```js
const sym =

```

---

Need:

```js

```

# Part 10 — Arrays vs Objects

---

Arrays:

```js
const arr = [1,2,3];
```

Internally:

```js
{
  0:1,
  1:2,
  2:3,
  length:3
}
```

---

Therefore:

```js
typeof []
```

↓

```js
"object"
```

---

# Part 11 — Sparse Arrays

---

Normal

```js
[1,undefined,3]
```

---

Sparse

```js
[1,,3]
```

---

Difference

```js
1 in arr
```

---

Undefined

```js
true
```

---

Sparse

```js
false
```

---

Map skips holes.

# Part 12 — Prototype Chain Traversal

---

Object lookup:

```js
obj.a
```

Searches:

```text
obj
↓
prototype
↓
prototype
↓
null
```

---

This is why:

```js
for...in
```

can see inherited properties.
